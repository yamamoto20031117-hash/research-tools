1. 概要・システム構成

QE 計算ランチャーは、ブラウザ上の Web UI から研究室の計算サーバー上の AiiDA / Quantum ESPRESSO にジョブを投入し、結果（バンド構造等）を自動取得するシステムです。

┌─────────────────┐        HTTPS         ┌──────────────────────────┐
│  ブラウザ (UI)   │  ◄───────────────►  │  計算サーバー (研究室)     │
│  qe-calculator   │      REST API        │  AiiDA + QE + AiiDAlab   │
└─────────────────┘     Bearer Token      └──────────────────────────┘

通信は Bearer トークン認証付きの HTTPS で保護されます。計算サーバーは研究室の高性能 PC（Linux）を想定しています。

2. 前提条件

項目要件
OSUbuntu 22.04+ / CentOS 8+ (Linux)
Python3.9 以上

ネットワーク外部からアクセス可能なポート（443 推奨）

3. AiiDA + AiiDAlab のインストール

A 方法 1: Docker（推奨）

AiiDAlab の公式 Docker イメージを利用する方法が最も簡単です。

# Docker イメージの取得と起動
docker pull aiidalab/aiidalab-docker-stack:latest

docker run -d \
  --name aiidalab \
  -p 8888:8888 \
  -p 5000:5000 \
  -v aiidalab_data:/home/jovyan \
  --restart unless-stopped \
  aiidalab/aiidalab-docker-stack:latest
ポート 8888 は JupyterLab UI、ポート 5000 は AiiDA REST API に使用します。

B 方法 2: ネイティブインストール

# 仮想環境作成
python3 -m venv ~/aiida-env
source ~/aiida-env/bin/activate

# AiiDA コアインストール
pip install aiida-core aiida-quantumespresso

# RabbitMQ と PostgreSQL のインストール
sudo apt install rabbitmq-server postgresql postgresql-client

# AiiDA プロファイル作成
verdi quicksetup --profile default \
  --db-backend core.psql_dos \
  --email your@email.com

# デーモン起動
verdi daemon start

4. Quantum ESPRESSO のセットアップ

インストール

# apt でインストール（Ubuntu）
sudo apt install quantum-espresso

# または ソースからビルド
wget https://github.com/QEF/q-e/releases/download/qe-7.3.1/qe-7.3.1-ReleasePack.tar.gz
tar xzf qe-7.3.1-ReleasePack.tar.gz
cd qe-7.3.1
./configure --enable-parallel
make all
sudo make install

AiiDA に計算機（Computer）を登録

# ローカルマシンを計算機として登録
verdi computer setup --label localhost \
  --hostname localhost \
  --transport core.local \
  --scheduler core.direct \
  --work-dir /tmp/aiida_run \
  --mpirun-command "mpirun -np {tot_num_mpiprocs}"

verdi computer configure core.local localhost
verdi computer test localhost

QE コードの登録

# pw.x の登録
verdi code create core.code.installed \
  --label pw-7.3 \
  --computer localhost \
  --default-calc-job-plugin quantumespresso.pw \
  --filepath-executable $(which pw.x)

# bands.x の登録
verdi code create core.code.installed \
  --label bands-7.3 \
  --computer localhost \
  --default-calc-job-plugin quantumespresso.bands \
  --filepath-executable $(which bands.x)

擬ポテンシャルのインストール

# SSSP ライブラリ（推奨）

aiida-pseudo install sssp --version 1.3 --functional PBE

5. REST API の有効化

AiiDA REST API の起動

# デフォルト設定で起動（ポート 5000）
verdi restapi --host 0.0.0.0 --port 5000

# バックグラウンドで起動する場合
nohup verdi restapi --host 0.0.0.0 --port 5000 &> /var/log/aiida-rest.log &

systemd サービス化（推奨）

# /etc/systemd/system/aiida-rest.service
[Unit]
Description=AiiDA REST API
After=network.target postgresql.service rabbitmq-server.service

[Service]
Type=simple
User=aiida
Environment="PATH=/home/aiida/aiida-env/bin:/usr/bin"
ExecStart=/home/aiida/aiida-env/bin/verdi restapi --host 127.0.0.1 --port 5000
Restart=always
RestartSec=5

[Install]
WantedBy=multi-user.target

sudo systemctl enable aiida-rest
sudo systemctl start aiida-rest

API 動作確認

# エンドポイント一覧
curl http://localhost:5000/api/v4

# 計算機一覧
curl http://localhost:5000/api/v4/computers

# ノード一覧
curl http://localhost:5000/api/v4/nodes?limit=5

6. セキュリティ設定

⚠️ AiiDA REST API をインターネットに直接公開しないでください。必ず nginx 等のリバースプロキシ経由で Bearer トークン認証を設けてください。

1 SSL 証明書の取得（Let's Encrypt）

sudo apt install certbot python3-certbot-nginx
sudo certbot --nginx -d calc.your-domain.com

2 Bearer トークンの生成

# ランダムな 64 文字のトークンを生成
openssl rand -hex 32
# 例: a1b2c3d4e5f6...（このトークンを安全に保管）

3 nginx リバースプロキシ設定

# /etc/nginx/sites-available/aiida-api
server {
    listen 443 ssl http2;
    server_name calc.your-domain.com;

    ssl_certificate     /etc/letsencrypt/live/calc.your-domain.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/calc.your-domain.com/privkey.pem;

    # Bearer トークン認証
    set $expected_token "Bearer YOUR_GENERATED_TOKEN_HERE";

    location /api/ {
        # トークン検証
        if ($http_authorization != $expected_token) {
            return 401 '{"error": "Unauthorized"}';
        }

        # CORS ヘッダー
        add_header 'Access-Control-Allow-Origin' 'https://your-research-tools-site.com' always;
        add_header 'Access-Control-Allow-Methods' 'GET, POST, OPTIONS' always;
        add_header 'Access-Control-Allow-Headers' 'Authorization, Content-Type' always;

        if ($request_method = 'OPTIONS') {
            add_header 'Access-Control-Allow-Origin' 'https://your-research-tools-site.com';
            add_header 'Access-Control-Allow-Methods' 'GET, POST, OPTIONS';
            add_header 'Access-Control-Allow-Headers' 'Authorization, Content-Type';
            add_header 'Access-Control-Max-Age' 86400;
            return 204;
        }

        # AiiDA REST API にプロキシ
        proxy_pass http://127.0.0.1:5000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}

sudo ln -s /etc/nginx/sites-available/aiida-api /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl reload nginx

4 ファイアウォール設定

# 443 のみ外部公開（5000 は localhost のみ）
sudo ufw allow 443/tcp
sudo ufw deny 5000/tcp
sudo ufw enable

7. QE 計算ランチャーとの接続

  1. QE 計算ランチャーを開き、右上の 接続設定 ボタンをクリック
  2. API エンドポイント に https://calc.your-domain.com/api/v4 を入力
  3. Bearer トークン に Step 6-2 で生成したトークンを入力
  4. 「接続テスト」ボタンで疎通確認 → 緑の「接続済み」表示になれば成功
接続情報はブラウザの localStorage に保存されます。トークンは暗号化されずに保存されるため、共有 PC での利用には注意してください。

8. ワークフロー解説

QE 計算ランチャーは以下の 4 段階パイプラインで計算を実行します：

ステップ内容QE コマンド
1. Relax構造最適化（原子位置 + セルパラメータ）pw.x (calculation='vc-relax')
2. SCF自己無撞着場計算pw.x (calculation='scf')
3. NSCF非自己無撞着場計算（バンド k 点）pw.x (calculation='nscf')
4. Bandsバンド構造の後処理bands.x

CIF ファイルの投入

TMD 結晶構造ビューアで作成した CIF ファイルをドラッグ＆ドロップまたはファイル選択で投入します。CIF から以下の情報を自動抽出します：

  • 格子定数（a, b, c, α, β, γ）
  • 空間群
  • 原子座標（fractional coordinates）
  • 元素種

計算設定

  • 擬ポテンシャル: SSSP Efficiency / Precision（サーバーにインストール済みのもの）
  • カットオフエネルギー: ecutwfc（波動関数）/ ecutrho（電荷密度）
  • k 点メッシュ: Monkhorst-Pack グリッド
  • スメアリング: Marzari-Vanderbilt / Methfessel-Paxton
  • 収束閾値: conv_thr（SCF 収束判定）

高度な設定（オプション）

  • スピン分極: nspin=2 で磁性計算
  • スピン軌道相互作用（SOC）: fully-relativistic 擬ポテンシャル使用
  • Hubbard U: DFT+U 補正
  • vdW 補正: DFT-D3 / vdW-DF2（層状物質に重要）

9. トラブルシューティング

接続テストが失敗する

  • CORS エラー: nginx の CORS 設定で Origin が正しいか確認
  • 401 Unauthorized: トークンが正しいか確認（Bearer プレフィックス不要 — UI が自動付与）
  • 502 Bad Gateway: AiiDA REST API が起動しているか sudo systemctl status aiida-rest で確認
  • タイムアウト: ファイアウォールでポート 443 が開いているか確認

計算が開始されない

  • AiiDA デーモンが動作しているか: verdi daemon status
  • RabbitMQ が起動しているか: sudo systemctl status rabbitmq-server
  • 計算機が正しく登録されているか: verdi computer test localhost

計算が失敗する

  • ログ確認: verdi process report <PK>
  • 出力ファイル確認: verdi calcjob outputcat <PK>
  • 擬ポテンシャルが存在するか: aiida-pseudo list
  • メモリ不足: k 点を減らすか ecutwfc を下げてテスト

Docker 環境の再起動

docker restart aiidalab
# ログ確認
docker logs -f aiidalab --tail 50
